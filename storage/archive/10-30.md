## 📋 YAML FILE REFERENCE (Quick Lookup)

### YAML Configuration Files in use:

| File | Purpose | Sections | Loaded By |
|------|---------|----------|-----------|
| **gamedev.yaml** | Game dev page playlists | featured_section, martian_games_playlists, learning_section, archive_games, grid_config | youtube-grid.js |
| **gaming.yaml** | Gaming page playlists | All gaming playlists with genre/platform tags, show_single_video: true | youtube-grid.js |
| **gamejams.yaml** | Game jam page playlists | Game jam highlights, Ludum Dare collections | youtube-grid.js |
| **diy.yaml** | DIY page playlists | Fashion, hair, nails, brand collabs, creative tech sections | youtube-grid.js |
| **blog-posts.yaml** | Blog metadata | Blog post titles, dates, excerpts (not YouTube related) | blog.php |

### Key YAML Properties to Know:

**Section-Level Properties:**
- `title` - Section heading
- `description` - Section description
- `icon` - Bootstrap icon name
- `columns` - Column count (overrides grid_config)
- `playlists` - Array of playlist objects

**Playlist-Level Properties:**
- `id` - YouTube playlist ID
- `title` - Playlist title
- `icon` - Bootstrap icon name
- `badge` - Badge text (Featured, Game Project, Martian Games)
- `description` - Short description
- `tags` - Array of tag slugs for filtering
- `page` - Internal page URL (e.g., "/game/purgatoryfell.php")
- `external_url` - External link (GameJolt, Steam, etc.)

**Grid Config Properties:**
- `{section}_columns` - Column count for specific section (e.g., `featured_columns: 3`)
- `columns` - Global column count (fallback)
- `videos_per_playlist` - Videos to display per playlist
- `enable_hover_effects` - Enable hover animations
- `enable_lazy_loading` - Lazy load video thumbnails
- `show_single_video` - Show only first video (gaming page style)

### YAML Loading Flow:
```
1. Page loads (e.g., gamedev.php)
   ↓
2. youtube-grid.js calls: YouTubeGrid.loadPageConfig('gamedev')
   ↓
3. Fetches: /resources/playlists/gamedev.yaml
   ↓
4. Uses js-yaml library: jsyaml.load(yamlText)
   ↓
5. Parses sections: featured_section, martian_games_playlists, etc.
   ↓
6. Creates HTML containers: #featured-playlists, #martian_games-playlists
   ↓
7. Renders playlist cards with tags: <div class="content-item" data-tags="vr,unity">
   ↓
8. tag-system.js reads data-tags attributes for filtering
```
