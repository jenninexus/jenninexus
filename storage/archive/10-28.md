## 📊 YOUTUBE CACHE SYSTEM EXPLAINED

### Remote Server Cache Files (21 files):
```
/var/www/jenninexus/storage/cache/youtube/*.json
```
- **What:** Server-side RSS feed cache (10-minute TTL)
- **Filename:** MD5 hash of RSS URL (e.g., `a1c302444120.json`)
- **Purpose:** Avoid hitting YouTube RSS feeds repeatedly
- **Auto-created:** By `get-youtube.php` proxy on first request
- **Safe to delete:** Yes - will regenerate on next page load

### How It Works:
```
Browser → youtube-grid.js → get-youtube.php proxy
                                    ↓
                          Check cache (10 min TTL)
                                    ↓
                          Fresh? Return cached JSON
                          Stale? Fetch new RSS from YouTube
                                    ↓
                          Parse XML → JSON with media:thumbnail
                                    ↓
                          Cache to storage/cache/youtube/
                                    ↓
                          Return to browser (24hr client cache)
```

### What Changed Today:
✅ **youtube-grid.js v3.0** now extracts **REAL thumbnails from RSS**:
- Before: Constructed generic URLs (`mqdefault.jpg`)
- After: Uses actual `<media:thumbnail>` URLs from RSS feed (`hqdefault.jpg` - higher quality!)
- Benefit: More reliable, higher quality thumbnails

## 🔧 TECHNICAL REFERENCE (Keep for Future Sessions)

### YouTube Cache Files Explained

**Filename Pattern:** `MD5(RSS_URL).json`
Example: `a1c302444120.json` = MD5 of RSS feed URL

**Contents:** JSON parsed from YouTube RSS XML
```json
{
  "link": {"@attributes": {"href": "..."}},
  "title": "Playlist Title",
  "entry": [
    {
      "id": "yt:video:VIDEO_ID",
      "title": "Video Title",
      "media:group": {
        "media:thumbnail": {
          "@attributes": {"url": "https://i3.ytimg.com/vi/VIDEO_ID/hqdefault.jpg"}
        }
      }
    }
  ]
}
```

**Lifecycle:**
1. First request → get-youtube.php fetches RSS from YouTube
2. Parse XML → JSON, extract video IDs + thumbnails
3. Cache to `storage/cache/youtube/HASH.json`
4. Return to browser
5. Next request within 10 min → serve from cache
6. After 10 min → fetch fresh RSS, update cache

**Safe Operations:**
- ✅ Delete cache files (will regenerate)
- ✅ Exclude from deploy (server creates own)
- ✅ Commit to git (.gitignore excludes storage/)
- ❌ Don't manually edit (auto-generated)
