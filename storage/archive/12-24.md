## 🔐 Patreon VIP Integration Quick Reference

**Purpose:** Client-side content gating with blur-to-reveal mechanism for VIP/patron-only content.

**How it works:**
1. VIP content starts blurred with `.vip-blur` class (`backdrop-filter: blur(8px)`)
2. JavaScript checks authorization via `/resources/api/check-patreon-membership.php`
3. If `is_patron: true`, the `revealVIP()` function removes blur classes
4. Content becomes interactive and visible to authorized patrons

**What's gated (Patreon page):**
- Patreon API Posts (`#vipGrid`)
- VIP YouTube Playlists (`#vip-playlists` from `patreon.yaml`)
- CC4 Guide PDF Preview & Download (`#cc4Preview`, `#cc4PdfLink`)
- Member Downloads Section (`#vipDownloads`)

**Setting up for another website:**
1. **Server Secrets** (`/var/www/yoursite/storage/secrets/patreon.json`):
   - `PATREON_CLIENT_ID` - OAuth app client ID
   - `PATREON_CLIENT_SECRET` - OAuth app secret
   - `PATREON_REDIRECT_URI` - Callback URL (e.g., `https://yoursite.com/patreon-callback.php`)
   - `PATREON_CREATOR_ACCESS_TOKEN` - Long-lived creator token
   - `PATREON_CAMPAIGN_ID` - Your campaign ID
   - `PATREON_WEBHOOK_SECRET` - Webhook HMAC secret

2. **CSS Classes** (copy to your theme CSS):
   - `.vip-blur` - Applies blur filter
   - `.vip-revealed` - Removes blur (authorized state)
   - `.vip-guest-view` - Guest-specific container styling

3. **JavaScript** (`patreon-auth-enhanced.js`):
   - `checkPatreonMembership()` - Calls API endpoint
   - `revealVIP()` - Removes blur, shows content
   - `hideVIP()` - Re-blurs for guests

4. **API Endpoints** (PHP):
   - `patreon-auth-start.php` - Initiates OAuth
   - `patreon-callback.php` - Handles OAuth callback
   - `/resources/api/check-patreon-membership.php` - Checks patron status
   - `/resources/api/get-patreon-posts.php` - Fetches posts
   - `/resources/api/get-patreon-tiers.php` - Fetches campaign tiers

5. **Storage Structure** (server, excluded from deployment):
   ```
   storage/
   ├── secrets/patreon.json (600, owner-only)
   ├── patreon/
   │   ├── tokens.json (runtime OAuth tokens)
   │   ├── user.json (runtime user identity)
   │   ├── posts_cache.json (cached posts)
   │   ├── tiers_cache.json (cached tiers)
   │   ├── logs/ (Patreon API logs)
   │   └── webhooks/ (webhook payloads)
   ```

**Full Documentation:** See `storage/docs/PATREON.md` for complete setup instructions, security requirements, and troubleshooting.

# YouTube Playlist Display Consistency - Dec 25, 2025

## 🎯 Overview
Standardized YouTube playlist/video display formats across JenniNexus pages for consistent user experience and maintainable codebase.

---

## 📋 Display Formats

### Format 1: Single-Card Layout (Default)
**Used by:** `youtube.php`, `gamedev.php`, `gaming.php`, `diy.php`

**Purpose:** Display multiple playlists with one video preview per playlist card

**Configuration:**
```yaml
layout: "single-card"
show_single_video: true
videos_per_playlist: 1
columns:
  xs: 1    # Mobile: 1 column (stacked)
  sm: 2    # Tablet: 2 columns
  md: 3    # Desktop: 3 columns
  lg: 3    # Large: 3 columns
```

**Usage:**
```javascript
window.YouTubeGrid.loadPageConfig('pagename');
```

**Example Pages:**
- `/gamedev.php` - Featured game projects (3-column grid of playlist cards)
- `/gaming.php` - Gaming playlists (3-column grid)
- `/youtube.php` - Featured playlists across all categories (3-column grid)
- `/diy.php` - DIY tutorial playlists (3-column grid)

---

### Format 2: Multi-Video Layout
**Used by:** `music.php`, `game/catgame.php`, `game/jennistyles.php`, `game/botborgs.php` (detail pages)

**Purpose:** Show multiple videos from a single playlist or section

**Configuration:**
```yaml
layout: "multi-video"
show_single_video: false
videos_per_playlist: 4-6
columns:
  xs: 1    # Mobile: 1 column (stacked)
  sm: 2    # Tablet: 2 columns (2x2 or 2x3)
  md: 4    # Desktop: 4 columns (1x4 row)
  lg: 4    # Large: 4 columns
```

**Usage (catgame.php example):**
```javascript
window.YouTubeGrid.renderPlaylist('catgame-videos', playlistId, {
  layout: 'multi-video',
  videos_per_playlist: 4,
  columns: { xs: 1, sm: 2, md: 4, lg: 4 },
  aspect_ratio: '16:9',
  show_playlist_title: false,
  enable_hover_effects: true,
  lazy_load: true
});
```

**Example Pages:**
- `/music.php` - Shows 6 videos per playlist (DJ tutorials, FL Studio, etc.)
- `/game/catgame.php` - Shows 4 Cat-As-Trophy gameplay videos in 1x4 row

---

## 🔧 Implementation Patterns

### Pattern A: Page-Level Config (Recommended)
**For:** Main section pages (gamedev, gaming, diy, youtube)

1. Create/update YAML config in `public_html/resources/playlists/{page}.yaml`
2. Define sections with `featured_section`, `learning_section`, etc.
3. Load via JavaScript:
   ```javascript
   window.YouTubeGrid.loadPageConfig('pagename');
   ```

**Benefits:**
- Centralized playlist management
- Easy to update without touching PHP/JS
- Consistent grid behavior across sections

---

### Pattern B: Direct Playlist Render
**For:** Game detail pages, special cases

Use when showing videos from ONE specific playlist:
```javascript
window.YouTubeGrid.renderPlaylist(containerId, playlistId, options);
```

**Example (catgame.php):**
```javascript
window.YouTubeGrid.renderPlaylist('catgame-videos', 'PLLzfSGTgy8YBL97yf_JRzcc3NWwk8FXyN', {
  layout: 'multi-video',
  videos_per_playlist: 4,
  columns: { xs: 1, sm: 2, md: 4, lg: 4 },
  aspect_ratio: '16:9'
});
```

## 📐 Responsive Grid Guidelines

### Bootstrap 5.3.8 Breakpoints
- **xs**: < 576px (Mobile phones - iPhone 390px, etc.)
- **sm**: ≥ 576px (Small tablets)
- **md**: ≥ 768px (Tablets, iPad)
- **lg**: ≥ 992px (Desktop, laptop)
- **xl**: ≥ 1200px (Large desktop)

### Standard Column Configurations

**3-Column Grid (Default for playlist cards):**
```yaml
columns: { xs: 1, sm: 2, md: 3, lg: 3 }
```
- Mobile: Stacked (1 col)
- Tablet: 2 columns
- Desktop: 3 columns

**4-Column Row (For video grids):**
```yaml
columns: { xs: 1, sm: 2, md: 4, lg: 4 }
```
- Mobile: Stacked (1 col)
- Tablet: 2x2 grid
- Desktop: 1x4 row

## 🚀 Best Practices

### 1. When to use Single-Card Layout
✅ Main section pages with multiple playlists  
✅ Landing pages showcasing variety of content  
✅ Tag/category pages with filtered playlists  

### 2. When to use Multi-Video Layout
✅ Game detail pages (show 4-6 gameplay videos)  
✅ Music production pages (show tutorial series)  
✅ Deep-dive content pages (explore single playlist)  

### 3. Responsive Design
✅ Always define all breakpoints (xs, sm, md, lg)  
✅ Mobile-first: Start with 1 column (xs: 1)  
✅ Test on iPhone 390px, iPad 768px, Desktop 1920px  

### 4. Performance
✅ Use `lazy_load: true` for below-fold content  
✅ Limit `videos_per_playlist` to 4-6 (avoid excessive API calls)  
✅ Enable hover effects only on desktop (`enable_hover_effects: true`)
