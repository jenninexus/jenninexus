### **PRIORITY 3: File Watcher / Auto-Build Setup** 🟢 ENHANCEMENT

#### Issue: No automatic CSS/JS rebuild on file changes
- Currently must run `scripts/build.ps1` manually after editing `src/assets/`
- Dev server (`dev-server.ps1`) does NOT watch for file changes

#### Proposed Solution: Add File Watcher Task
**Create:** `scripts/watch.ps1` (NEW FILE)
```powershell
#!/usr/bin/env pwsh
<#
.SYNOPSIS
    JenniNexus File Watcher - Auto-rebuild on src/assets changes
.DESCRIPTION
    Watches src/assets/ directory for changes and runs build.ps1 automatically
    Useful for CSS/JS development without manual builds
#>

param(
    [int]$Port = 8002,
    [switch]$NoBrowser
)

$ProjectRoot = Split-Path -Parent $PSScriptRoot
$WatchPath = Join-Path $ProjectRoot "src\assets"
$BuildScript = Join-Path $PSScriptRoot "build.ps1"

Write-Host "═══════════════════════════════════════════════════════" -ForegroundColor Cyan
Write-Host "  JenniNexus File Watcher" -ForegroundColor Cyan
Write-Host "═══════════════════════════════════════════════════════" -ForegroundColor Cyan
Write-Host ""
Write-Host "👀 Watching: $WatchPath" -ForegroundColor Yellow
Write-Host "🔨 Build Script: $BuildScript" -ForegroundColor Cyan
Write-Host "🌐 Dev Server: http://localhost:$Port" -ForegroundColor Green
Write-Host ""

# Start dev server in background job
$devServerJob = Start-Job -ScriptBlock {
    param($Script, $Port)
    & $Script -Port $Port
} -ArgumentList "$PSScriptRoot\dev-server.ps1", $Port

Write-Host "✅ Dev server started (Job ID: $($devServerJob.Id))" -ForegroundColor Green

# Open browser (optional)
if (-not $NoBrowser) {
    Start-Sleep -Seconds 2
    Start-Process "http://localhost:$Port"
}

# Create file watcher
$watcher = New-Object System.IO.FileSystemWatcher
$watcher.Path = $WatchPath
$watcher.IncludeSubdirectories = $true
$watcher.EnableRaisingEvents = $true

# Watch for CSS, JS, YAML changes
$watcher.Filter = "*.*"

# Debounce mechanism (prevent multiple builds from rapid saves)
$script:lastBuild = [DateTime]::MinValue
$debounceSeconds = 2

$onChanged = {
    param($source, $e)
    
    # Only build for relevant file types
    $ext = [System.IO.Path]::GetExtension($e.Name)
    if ($ext -notin @('.css', '.js', '.yaml', '.json', '.svg', '.png', '.jpg', '.jpeg')) {
        return
    }
    
    # Debounce check
    $now = [DateTime]::Now
    if (($now - $script:lastBuild).TotalSeconds -lt $debounceSeconds) {
        return
    }
    $script:lastBuild = $now
    
    Write-Host "`n🔄 Change detected: $($e.Name)" -ForegroundColor Yellow
    Write-Host "🔨 Running build..." -ForegroundColor Cyan
    
    try {
        & $BuildScript
        Write-Host "✅ Build complete!" -ForegroundColor Green
    } catch {
        Write-Host "❌ Build failed: $($_.Exception.Message)" -ForegroundColor Red
    }
}

# Register event handlers
Register-ObjectEvent -InputObject $watcher -EventName Changed -Action $onChanged | Out-Null
Register-ObjectEvent -InputObject $watcher -EventName Created -Action $onChanged | Out-Null
Register-ObjectEvent -InputObject $watcher -EventName Renamed -Action $onChanged | Out-Null

Write-Host "✅ File watcher active - edit files in src/assets/ to auto-build" -ForegroundColor Green
Write-Host ""
Write-Host "Press Ctrl+C to stop watcher and dev server..." -ForegroundColor Yellow
Write-Host ""

# Keep script running
try {
    while ($true) {
        Start-Sleep -Seconds 1
        
        # Check if dev server job is still running
        if ($devServerJob.State -ne 'Running') {
            Write-Host "❌ Dev server stopped unexpectedly" -ForegroundColor Red
            break
        }
    }
} finally {
    # Cleanup
    Write-Host "`n🛑 Stopping file watcher..." -ForegroundColor Yellow
    $watcher.EnableRaisingEvents = $false
    $watcher.Dispose()
    
    Write-Host "🛑 Stopping dev server..." -ForegroundColor Yellow
    Stop-Job -Job $devServerJob
    Remove-Job -Job $devServerJob
    
    Write-Host "✅ Cleanup complete" -ForegroundColor Green
}
```

## 📚 BUILD SYSTEM CLARIFICATION

### Question: Does build.ps1 handle CSS AND JS?
**Answer:** YES! `build.ps1` handles both:

**CSS Pipeline:**
1. Copies from `src/assets/css/` → `public_html/resources/css/`
2. Only copies if source is newer than destination (incremental builds)
3. Minifies CSS to `.min.css` using `npx clean-css-cli`
4. Skips minification if `.min.css` is up-to-date
5. **Average savings:** 62% file size reduction (~40KB+)

**JavaScript Pipeline:**
1. Copies from `src/assets/js/` → `public_html/resources/js/`
2. Only copies if source is newer than destination
3. **Note:** JS minification is NOT automatic in build.ps1
4. **Use build-all.ps1 for JS minification** (calls optimize-assets.ps1)

**Other Assets:**
- Fonts: `src/assets/fonts/` → `public_html/resources/fonts/`
- SVGs: `src/assets/svgs/` → `public_html/resources/svgs/`
- PDFs: `src/assets/pdfs/` → `public_html/resources/pdfs/`
- Images: `src/assets/images/` → `public_html/resources/images/`
- Playlists: `src/assets/playlists/*.yaml` → `public_html/resources/playlists/`
- JSON configs: `src/assets/*.json` → `public_html/resources/playlists/`

### build.ps1 vs build-all.ps1

**build.ps1** (Fast, incremental):
- Copies changed files only
- Minifies CSS (clean-css-cli)
- ~5-10 seconds
- **Use for:** Quick CSS/JS development iterations

.\scripts\watch.ps1

### Troubleshooting

**Watcher not triggering?**
- Check if you're editing files in `src/assets/` (not `public_html/resources/`)
- Verify 2-second debounce hasn't blocked your change
- Check terminal output for errors

**Dev server not starting?**
- Check if port 8002 is already in use: `Get-Process -Id (Get-NetTCPConnection -LocalPort 8002).OwningProcess`
- Try different port: `.\scripts\watch.ps1 -Port 3000`

**Cache not working?**
- Check directory exists: `Test-Path storage\cache\youtube`
- Check permissions (should be writable)
- Check get-youtube.php error logs

**Build fails?**
- Check if npm is installed: `npm -v`
- Check if clean-css-cli is installed: `npx clean-css-cli -v`
- Install if missing: `npm install -g clean-css-cli`
